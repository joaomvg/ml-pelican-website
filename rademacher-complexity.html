"Rademacher complexity"

Joao Gomes

 Sat 02 May 2020
Category: Machine Learning
Tags: data science

The empirical Rademacher complexity of a hypothesis class \(G=\{g\}\) is defined as an average over the training set \(S=(z_1,\ldots,z_m)\) in the following way:

$$\hat{\mathcal{R}}(G)=E_{\sigma}\left[\text{sup}_{g\in G}\frac{1}{m}\sum_{i=1}^{m}\sigma_i g(z_i)\right]$$

where \(\sigma_i\) are \(m\) independently and uniformly distributed random variables in the interval \([-1,1]\). Since \(E(\sigma)=0\), we see that the above average is the correlation between \(\sigma\) and \(g(z)\). The Rademacher complexity therefore measures how well a hypothesis class correlates with noise. If a class has enough complexity, it will have better ability to correlate with noise and hence have higher rademacher complexity.

The Rademacher complexity, rather than the empirical one, is in turn defined as the statistical average over the true distribution \(D(z)\):

$$\mathcal{R}(G)=E_{\sim D^m}(\hat{\mathcal{R}}(G))$$

One can determine several bounds on the Rademacher complexity. One of particular interest considers the growth function. Remember that the growth function \(\Pi(m)\) is the maximal number of distinct ways of classifying a set of \(m\) points \(z_1,\ldots,z_m\) using an hypothesis class \(\mathcal{H}\). In order to calculate this bound we need the following lemma:

Massart's Lemma: let \(A\subset \mathbb{R}^m\) be a finite set, and \(r=\text{max}_{x\in A}\|x\|_2\), then

$$E_{\sigma}\left[\frac{1}{m}\text{sup}_{x\in A}\sum_i \sigma_ix_i\right]\leq \frac{r\sqrt{2\ln|A|}}{m}$$
